# 🤝 Contributing to Business Lead Generation AI

First off, **thank you for considering contributing!** 🎉
Your support helps make this project better, smarter, and more useful for everyone.

---

## 📌 How You Can Contribute

We welcome all types of contributions:

| Type | Description |
|------|-------------|
| 🐛 Bug Fixes | Help us squash bugs and improve stability |
| ✨ New Features | Suggest and build new functionality |
| 📚 Documentation | Improve guides, README, or inline comments |
| ⚡ Performance | Optimize speed, efficiency, or API usage |
| 🧪 Testing & Feedback | Report issues or test existing features |

---

## 🚀 Getting Started

### 1. 🍴 Fork the Repository
Click the **Fork** button on the top right of this repo.

### 2. 📥 Clone Your Fork

```bash
git clone https://github.com/nikhil-ai-insights/business-lead-generation-ai.git
```

### 3. 🌿 Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

### 4. ✏️ Make Your Changes

- Follow clean coding practices
- Keep changes minimal and focused
- Add comments where necessary

### 5. 💾 Commit Your Changes

```bash
git commit -m "Add: short description of your feature"
```

### 6. 📤 Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 7. 🔁 Create a Pull Request

- Go to the original repo
- Click **New Pull Request**
- Add a clear description of your changes

---

## 📏 Contribution Guidelines

- ✔ Keep pull requests small and focused
- ✔ Write clear and meaningful commit messages
- ✔ Follow existing project structure
- ✔ Avoid breaking existing functionality
- ✔ Test your changes before submitting

---

## 🧪 Reporting Bugs

If you find a bug:

1. Check if it's [already reported](../../issues)
2. If not, [create a new issue](../../issues/new)
3. Include the following details:

```
- Description of the problem
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
```

---

## 💡 Feature Requests

Have an idea? We'd love to hear it!

1. [Open an issue](../../issues/new)
2. Clearly describe your idea
3. Explain the benefit or use case

---

## 📂 Project Structure

Make sure your contributions align with the overall workflow structure:

```
Lead Capture → Search Builder → Data Extraction → Filtering → Formatting → Storage
```

Each stage is modular — contributions should respect the boundaries of each step and avoid tightly coupling components.

---

## 🔐 Security

If you discover a **security vulnerability**, please do **NOT** open a public issue.

> 📧 Contact the maintainer directly via GitHub: [@nikhil-ai-insights](https://github.com/nikhil-ai-insights)

We take security seriously and will respond promptly.

- Be **respectful** and **constructive** in all interactions
- Welcome and support contributors of all experience levels
- Provide helpful, thoughtful feedback on pull requests
- We aim to build a **welcoming and collaborative** environment for everyone
